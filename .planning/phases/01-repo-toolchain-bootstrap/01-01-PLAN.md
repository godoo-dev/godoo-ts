## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| developer workstation → npm registry | pnpm install fetches packages from registry; postinstall scripts run with local privileges |
| developer workstation → GitHub API | `gh` CLI makes authenticated API calls to create repo and (later) rulesets |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | npm package install (pnpm install) | mitigate | All packages in RESEARCH.md Package Legitimacy Audit are slopcheck-verified Approved; no [ASSUMED] or [SUS] remain after false-positive confirmation for vitest |
| T-01-02 | Tampering | pnpm-lock.yaml | mitigate | lockfile-discipline rule: commit pnpm-lock.yaml with package.json in same commit; CI uses --frozen-lockfile to detect drift |
| T-01-03 | Information Disclosure | GITHUB_TOKEN / gh credentials | mitigate | gh auth status check only verifies auth state — never reads token value; no secrets committed to repo files |
| T-01-04 | Elevation of Privilege | force-push to main | mitigate | Branch protection ruleset (Plan 03) adds non_fast_forward rule to main; Plan 01 pushes to develop only, not main |
| T-01-SC | Tampering | slopcheck gate for npm installs | mitigate | Package Legitimacy Audit in RESEARCH.md confirms all packages Approved; no blocking checkpoint needed |
</threat_model>
