<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner → npm registry | CI uses pnpm install --frozen-lockfile; no new package fetches unless lockfile changes |
| developer → GitHub REST API (rulesets) | gh CLI makes authenticated POST to create branch protection; scope must include repo admin |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Elevation of Privilege | Force-push to main | mitigate | non_fast_forward rule added to branch ruleset (RESEARCH.md Security Domain recommendation) |
| T-03-02 | Tampering | Lockfile drift in CI | mitigate | CI uses `pnpm install --frozen-lockfile`; incompatible lockfile fails install with ERR_PNPM_LOCKFILE_BREAKING_CHANGE before any build |
| T-03-03 | Information Disclosure | GITHUB_TOKEN in CI | mitigate | GitHub Actions injects GITHUB_TOKEN as env var; not logged; not committed to repo; only consumed by `@changesets/changelog-github` at release time (Phase 3) |
| T-03-04 | Tampering | Branch ruleset bypass | accept | Solo maintainer project; ruleset requires CI pass; no required reviews by D-09 design; force-push blocked by non_fast_forward rule |
| T-03-SC | Tampering | slopcheck gate | mitigate | No new package installs in Plan 03; all packages installed in Plan 01 from verified audit |
</threat_model>
