## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | npm registry — stub package content | mitigate | Stub's index.js throws on require; `0.0.0` is below any `^x.y.z` semver constraint; no consumer can accidentally depend on it |
| T-03-02 | Spoofing | npm package name squatting (@godoo/*) | mitigate | Publishing stubs immediately claims the names; the @godoo org is already under Marc's control |
| T-03-03 | Information Disclosure | NPM_TOKEN_BREAKGLASS value | mitigate | Token stored as GitHub encrypted secret; only passed to workflow via ${{ secrets.NPM_TOKEN_BREAKGLASS }}; never echoed; rotate annually or on incident per D-03 |
| T-03-04 | Elevation of Privilege | Stub pushed inside pnpm workspace | mitigate | _stubs/ directory is outside packages/; pnpm-workspace.yaml glob is 'packages/*'; changesets cannot discover stubs |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in workspace for Phase 3 — package legitimacy audit in RESEARCH.md confirms no workspace-level installs |
