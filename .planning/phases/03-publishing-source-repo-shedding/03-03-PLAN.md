    Trigger: push to main ONLY (not develop, not pull_request)
    Concurrency: ${{ github.workflow }}-${{ github.ref }}

    Top-level permissions block (all three required):
    - contents: write  (changesets/action pushes version commits)
    - pull-requests: write  (changesets/action creates Version Packages PR)
    - id-token: write  (REQUIRED for npm OIDC trusted publishing)

    Single job named "release" with name "Version or Publish":
    - uses: actions/checkout@v4 with fetch-depth: 0  (required so changesets can detect changed packages)
    - uses: pnpm/action-setup@v6  (BEFORE setup-node — load-bearing order from ci.yml pattern)

    - run: pnpm install --frozen-lockfile
    - run: pnpm build
    - name: Create Release PR or Publish, id: changesets, uses: changesets/action@v1 with:
        publish: pnpm release  (maps to "release": "changeset publish" in root package.json)
        title: chore: version packages
        commit: chore: version packages

        GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        NPM_CONFIG_PROVENANCE: 'true'  (env-var form — cleaner than --provenance flag in changeset publish)
        NODE_AUTH_TOKEN: ''  (explicitly cleared — prevents setup-node token from shadowing OIDC; Pitfall 1)
        # No token fallback — D-03 dropped 2026-05-26; release auth is OIDC-only.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner → npm registry | OIDC token issued per-workflow-run; trusted publisher validates subject claim |
| develop branch → main branch | PR merge triggers release workflow; branch protection requires CI pass |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-09 | Tampering | NODE_AUTH_TOKEN shadowing OIDC | mitigate | Explicitly set NODE_AUTH_TOKEN: '' in changesets step env; documented in Pitfall 1 |
| T-03-10 | Elevation of Privilege | Workflow runs on Node 22 (npm v10 — no OIDC) | mitigate | Release workflow hard-pins node-version: 24 (single value, not matrix); never inherits ci.yml's 22+24 matrix |
| T-03-11 | Tampering | Changeset file with wrong package name causes wrong version bump | mitigate | Package names taken verbatim from .changeset/config.json; verified by grep gate in acceptance criteria |
| T-03-12 | Denial of Service | Release workflow triggers on develop push (not intended) | mitigate | Workflow trigger is push: branches: [main] only — no develop trigger |
| T-03-13 | Repudiation | No provenance attestation on published packages | mitigate | NPM_CONFIG_PROVENANCE: 'true' ensures SLSA attestation is attached to each published version |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed — changesets/action is a GitHub Action reference, not a workspace install |
</threat_model>
