## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions release workflow → npm registry | OIDC handshake; trusted publisher validates repo + workflow subject claim |
| Version Packages PR → main branch | changesets/action pushes version commit; branch protection CI must pass |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-14 | Tampering | workspace:* published as-is in testcontainers tarball | mitigate | pnpm's native publish resolves workspace:* to concrete versions; Task 2 explicitly verifies peerDependencies output (Pitfall 3 check) |
| T-03-15 | Repudiation | Published packages lack SLSA provenance | mitigate | NPM_CONFIG_PROVENANCE: 'true' is set in release.yml; provenance attestation visible via npm view @godoo-dev/client --json | jq '.dist.attestations' |
| T-03-16 | Elevation of Privilege | Break-glass token accidentally used as default | n/a | D-03 dropped 2026-05-26 — no token exists; workflow is OIDC-only with no token fallback. |
| T-03-17 | Denial of Service | Version Packages PR not created (changesets sees no changes) | mitigate | pnpm changeset status verified in Plan 03-03 Task 2; if no PR appears, check changeset status again and seed no-op patch changeset if needed |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No new workspace packages installed in this plan |
</threat_model>
